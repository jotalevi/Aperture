# Aperture

A fast, local, terminal-native image generation REPL for Apple Silicon. Load your model once, iterate at the prompt, full control over every parameter — with a background job queue and hot reload.

Aperture runs SDXL entirely on your Mac via Metal/MPS — nothing leaves the machine. It keeps your checkpoint resident across the session, so after the initial load you generate at the speed of typing.

## Features

- **Local & private** — runs entirely on your Mac, no cloud
- **Load once, generate many** — model stays in memory across the whole session
- **Background jobs** — queue prompts and keep typing while they render
- **Hot reload** — swap checkpoints without restarting the process
- **Full parameter control** — size, seed, steps, scheduler, guidance, batch count, all inline
- **Photoreal defaults** — low-CFG + DPM++ Karras tuning out of the box
- **Self-bootstrapping** — first run creates the venv and installs dependencies
- **Plain-text config** — token and defaults in a simple local file

## Requirements

- Apple Silicon Mac (M1 or newer)
- Python 3.12 recommended (3.13 has known issues with the ML stack)
- ~10 GB free disk for dependencies + a checkpoint

## Install & run

```bash
git clone <repo-url>
cd aperture
python main.py
```

First run creates `.venv`, installs dependencies, and writes `configs/config.txt`, then drops you into the shell. Subsequent runs launch straight in.

## Setup

Point Aperture at a photoreal checkpoint for best results — base SDXL looks synthetic. Good choices: **RealVisXL**, **Juggernaut XL**, **epiCRealism XL** (download the `.safetensors` from Hugging Face or Civitai).

From inside the shell:

```
>>> --conf.hf_token=hf_xxxxxxxxxxxxxxxx
>>> --conf.checkpoint=RealVisXL_V5.0.safetensors
>>> --conf.outdir=./output
>>> --conf.list
>>> --reload
```

`--reload` applies a new checkpoint without restarting (see below).

## Usage

Type a prompt and press Enter. Inline params go at the **start** of the line:

```
>>> woman by a cafe window, 35mm, natural skin texture, soft window light
>>> --q=4 --size=1024x1024 candid street photo, overcast light
>>> --sketchmode --q=6 testing compositions quickly
>>> --scheduler=eulera --steps=40 --filename=final close-up portrait, freckles
```

### Generation params

| Param | Default | Description |
|-------|---------|-------------|
| `--size=WxH` | `832x1216` | Output dimensions |
| `--seed=N` | from config | Base seed (increments per image in a batch) |
| `--steps=N` | `30` | Denoise steps |
| `--scheduler=NAME` | `dpmpp2m` | `dpmpp2m`, `dpmppsde`, `euler`, `eulera`, `unipc` |
| `--guidance=F` | `5.0` | CFG scale (lower = more natural) |
| `--q=N` | `1` | How many images to generate |
| `--filename=NAME` | timestamp | Output name (`_1.._N` suffix when `q>1`) |
| `--neg="..."` | built-in | Per-shot negative prompt override |
| `--sketchmode` | off | Fast draft: caps steps@12, cfg@3.0 |

Params persist as sticky defaults — you only specify what changes between shots.

## Background jobs

Activate background mode by typing `--jobs` on its own. While active, prompts you enter are **queued and generated by a worker thread** so you can keep typing the next one. A job list renders just above the input line:

```
  ── jobs ───────────────────────────────────────────
  [#1   | woman by a cafe win… | 1718…_1234.png        | done]
  [#2   | candid street photo… | final.png             |  47%]
  [#3   | close-up portrait,… | 1718…_99.png          | pending]
  ───────────────────────────────────────────────────
>>>
```

Each row is `[#id | prompt (sliced) | filename | state]`, where state is `pending`, a live `NN%`, `done`, or `error`.

| Command | Description |
|---------|-------------|
| `--jobs` | Toggle background mode on/off |
| `--jobs.clear` | Remove finished/errored jobs from the list |

Notes:
- A single worker processes jobs one at a time — generation is GPU-bound, so serializing keeps memory predictable on 16 GB machines.
- `--q=N` in jobs mode enqueues N separate jobs (seed-incremented), each tracked individually.
- Press Enter on an empty line to refresh the job list and watch progress tick.
- On exit (`Ctrl+C`), Aperture waits for the in-flight job to finish so you don't get a half-written PNG.

## Hot reload

`--reload` reloads `configs/config.txt` from disk and rebuilds the pipeline in place, so a changed `--conf.checkpoint` takes effect **without restarting the process**:

```
>>> --conf.checkpoint=Juggernaut_XL_v9.safetensors
>>> --reload
  loading Juggernaut_XL_v9.safetensors ...
  ready in 14.3s
  reload complete.
```

If background jobs are queued or running, `--reload` waits for them to drain first (the worker holds a reference to the live pipeline). On a 16 GB machine, swapping into a large checkpoint can briefly hold both models in memory during teardown — if you hit a memory error, a full process restart is the safe fallback.

### Config commands

| Command | Description |
|---------|-------------|
| `--conf.KEY=VALUE` | Set & persist a config value |
| `--conf.list` | Show current config (token masked) |
| `--conf.path` | Print config file location |

Keys: `hf_token`, `checkpoint`, `outdir`, `seed`, `steps`, `scheduler`, `guidance`, `default_size`

### Shell commands

| Command | Description |
|---------|-------------|
| `--help` / `--h` | Show the manual |
| `--reload` | Rebuild pipeline from current config |
| `Ctrl+C` | Exit (waits for the active job) |

## Tips for realism

- **Checkpoint matters most** — a photoreal fine-tune beats any amount of param tuning on base SDXL.
- **Keep CFG low** (4–5.5). High guidance creates the oversaturated "AI sheen."
- **Prompt like a photographer** — describe lens, lighting, and film stock rather than "beautiful."
- **Generate at native resolution** (832×1216 for portraits, 1024×1024 square) and upscale after.
- **Use `--sketchmode`** to find a composition fast, then re-run without it for the final.

## Performance (Apple Silicon)

The model loads once and stays resident, so per-image cost is just the diffusion run. Attention and VAE slicing are enabled by default to stay off the swap cliff on 16 GB machines. Expect roughly 25–45 s per 1024-class image on a base M4, faster at lower step counts. Background jobs don't make any single image faster — the GPU is the bottleneck — but they let you line up a session's worth of work without waiting at the prompt.
